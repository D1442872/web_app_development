# 產品需求文件 (PRD) - 活動報名系統

## 1. 專案概述

### 背景與動機
在學校或社團舉辦活動時，主辦方經常需要處理繁雜的報名流程，例如使用紙本或零散的表單工具。這不僅增加了管理負擔，也難以有效處理報名額滿與候補名單。因此，我們需要一個集中管理的「活動報名系統」，讓主辦方能輕鬆建立活動、管理名額，同時讓參與者能享有流暢的報名體驗。

### 目標用戶
- 學校社團幹部
- 活動主辦人
- 欲報名參加活動的學生（使用者）

### 核心價值主張
提供快速建立活動頁面、線上報名及自動候補機制，大幅簡化活動建立、管理報名名單與使用者報名的端到端流程。

---

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事（User Story）呈現：

1. **建立活動專頁**
   - **使用者故事**：作為[活動主辦人]，我希望[能夠設定活動名稱、時間、地點、內容描述及名額限制]，以便[讓學生了解活動詳情並吸引他們報名]。
2. **線上報名表單**
   - **使用者故事**：作為[想要參加活動的學生]，我希望[能夠填寫姓名、學號、連絡電話等資訊]，以便[快速完成活動報名]。
3. **報名進度查詢**
   - **使用者故事**：作為[已報名的學生]，我希望[能夠輸入學號查詢自己的報名狀態]，以便[確認自己是「已報名成功」還是處於「候補中」]。
4. **活動清單顯示**
   - **使用者故事**：作為[想要參加活動的學生]，我希望[能在首頁瀏覽所有正在招募中與已結束的活動列表]，以便[挑選我有興趣參與的活動]。
5. **報名額滿自動候補**
   - **使用者故事**：作為[活動主辦人]，我希望[當報名人數超過名額限制時，系統能自動將後續報名者轉入候補名單]，以便[省去手動管理候補名單的時間，並在有人取消時能依序遞補]。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask
- **前端模板**：使用 Jinja2
- **資料庫**：使用 SQLite

### 效能與安全考量
- **高併發處理**：考量熱門活動開放報名瞬間可能湧入大量流量，系統需具備基礎的防崩潰機制（例如資料庫 Transaction Lock 防止超賣）。
- **資訊安全**：妥善過濾並驗證使用者輸入的所有資料，防範 SQL Injection 與 XSS 等常見網路攻擊。
- **隱私保護**：妥善保管學生報名時所填寫的個資（姓名、學號、連絡電話），非授權人員無法輕易獲取。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級 | 功能說明 |
| :--- | :--- |
| **Must Have**<br>(核心必備) | - 建立活動專頁 (填寫活動基本資訊與名額)<br>- 線上報名表單<br>- 活動清單顯示 (首頁顯示所有活動) |
| **Should Have**<br>(強烈建議) | - 報名額滿自動候補機制<br>- 報名進度查詢功能 |
| **Nice to Have**<br>(加分項目) | - 報名/候補成功的 Email 通知<br>- 主辦後台可將報名名單匯出為 Excel/CSV<br>- 活動當天報到/簽到功能 |
